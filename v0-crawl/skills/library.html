Beacon Skill Library

Download a skill that applies to your role and run it in your own AI tool. No engineer in the loop — every skill grounds its output in real, clickable sources, the same way the exec brief does.

What this page is This is Beacon's answer to pain ⑥ — "can't hand it to a non-engineer." Each card is a self-contained skill (a folder with a SKILL.md) a teammate copies into their Claude/agent tool. It's the on-demand front door; the always-on, scheduled workflows (exec brief, deal decision-log) run in n8n. These are product skills — not to be confused with the engineering playbooks in the repo root.

Sales / BD / GTM

prospect-intel

Turn a prospect — a company, a person, or an onchain entity — into a grounded, citable one-page intel brief before a call, a deal review, or a qualification pass. Every factual line cites a real source; nothing is invented (the same trust contract as the exec brief).

What's real, not mocked: a read-only onchain footprint on Ethereum L1 + OP Mainnet — balances and activity from public RPCs, no keys, no transactions — so an onchain claim cites a block-explorer link a reviewer can click. For an OP Stack prospect, "do they actually transact, and where?" is qualifying signal.

▶ Open the skill (SKILL.md)

Install — copy the folder into your agent tool, then run the onchain step:

cp -r v0-crawl/skills/prospect-intel ~/.claude/skills/prospect-intel
cd v0-crawl && npm install
npm run prospect:onchain -- vitalik.eth      # or a 0x address

How a skill earns trust

Every skill in this library inherits Beacon's grounding contract — the same logic the n8n workflows enforce in v0-crawl/src/trust/ (tested, 5/5):

📎 Cite a real source — every factual claim links to something the researcher actually opened. No citation → it's an "unknown", not a fact.
🚫 Never invent a number — no funding figure, %, or multiplier unless it's in a cited source. The #1 way sales prep burns trust.
⛓️ Onchain is read, not guessed — balances/activity come from a real RPC read and cite a block-explorer URL.
🔒 No secrets / PII — the brief never carries a credential or a private email (the gate hard-fails on these).
