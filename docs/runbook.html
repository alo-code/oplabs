5. Q&A quick answers

QuestionAnswer
Why TypeScript, not Python/Go? Strongest language for a clean runnable slice; production stays polyglot behind seams. Judgment call, not religion.
Why n8n, not custom code? It removes 3 of 6 pains for free. The only custom code is the trust core — the differentiator. Build vs. compose judgment.
How does this earn trust in production? Gate fails closed (held ≠ published), evals from day one, run history is the audit log. Signals gate each stage, not dates.
Isn't 5 sources scope creep for Crawl? In n8n, a source is a node. The trust core is source-agnostic. More sources scale the brief, not the risk.
What if the model is down or wrong? Gate fails closed. Model is swappable behind an interface. n8n retries + run history make failures loud.
Why v0 and v1? v0 proves one workflow earns trust. v1 proves the platform thesis. They share the same trust spine — v0 wasn't rewritten, it was joined.
